## 🎯 关键决策记录

### 1. openclaw-china 使用官方版本
- ❌ 不使用本地修改版本（`openclaw-china-main/`）
- ✅ 使用官方 npm 包：`@openclaw-china/channels`
- 原因：保持基础镜像的通用性和稳定性

### 2. entrypoint.sh 不预构建
- ❌ 不包含在预构建镜像中
- ✅ 运行时从 HF 仓库复制
- 原因：保持配置的灵活性，方便快速修改

### 3. 补丁和资源运行时下载
- ❌ 不包含在预构建镜像中
- ✅ 运行时从 Supabase 下载
- 原因：这些文件经常变化，预构建会失去灵活性

## 📞 故障排查

### GitHub Actions 构建失败

检查：
1. Secrets 是否正确配置
2. Docker Hub Token 是否有效
3. 查看 Actions 日志定位错误

### HF 拉取镜像失败

检查：
1. 镜像是否已成功推送到 Docker Hub
2. `Dockerfile-hf` 中的镜像名称是否正确
3. Docker Hub 镜像是否设置为公开

### 运行时错误

检查：
1. HF Space 日志
2. Supabase 文件是否已上传
3. HF Secrets 是否正确配置
4. entrypoint.sh 是否正确执行
