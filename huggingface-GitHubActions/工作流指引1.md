# OpenClaw 预构建镜像使用指南

## 概述

为了加快 HuggingFace Spaces 的部署速度，我们使用 GitHub Actions 预构建基础镜像。

## 架构说明

### 预构建镜像包含（不常变化）
- ✅ OpenClaw 官方代码 + 依赖 + 构建产物
- ✅ ClawMate 基础插件
- ✅ openclaw-china 插件（官方版本）
- ✅ Chromium + 所有依赖库
- ✅ agent-browser + Playwright
- ✅ agent-browser skill
- ✅ 系统依赖和工具

### 运行时下载（经常变化）
- ❌ 自定义补丁（`patches/clawmate/*.ts`）
- ❌ 角色资源（`characters/maggie/*`）
- ❌ 配置文件（`openclaw.json`, `.env`）
- ❌ 系统提示（`AGENTS.md`, `TOOLS.md`）
- ❌ entrypoint.sh（运行时配置逻辑）

## 文件说明

### 1. Dockerfile-prebuilt
- **用途**：GitHub Actions 构建基础镜像
- **包含**：所有耗时的构建步骤
- **构建时间**：约 60 分钟（一次性）
- **触发条件**：
  - 推送到 main 分支
  - 修改 `openclaw-china-main/**`
  - 修改 `Dockerfile-prebuilt`
  - 手动触发

### 2. Dockerfile-hf
- **用途**：HuggingFace Spaces 使用
- **基于**：预构建镜像
- **构建时间**：5-10 分钟（只需拉取镜像）
- **包含**：entrypoint.sh 脚本

### 3. Dockerfile-supabase（保留）
- **用途**：原始完整构建方案（备用）
- **构建时间**：60 分钟
- **状态**：保留作为备份

### 4. entrypoint.sh-supabase（保留）
- **用途**：运行时配置和文件下载
- **不预构建**：保持灵活性
- **功能**：
  - 从 Supabase 下载 .env
  - 下载并覆盖补丁文件
  - 下载角色资源
  - 下载配置文件
  - 启动服务

## 使用步骤

### 第一步：设置 Docker Hub Secrets

在 GitHub 仓库设置中添加以下 Secrets：

1. `DOCKERHUB_USERNAME`：你的 Docker Hub 用户名
2. `DOCKERHUB_TOKEN`：你的 Docker Hub Access Token

获取 Access Token：
1. 登录 https://hub.docker.com
2. Account Settings → Security → New Access Token
3. 复制生成的 token

### 第二步：修改 Dockerfile-hf

将第一行的镜像名称改为你的：

```dockerfile
FROM viciy2023/openclaw-clawmate:latest
```

**注意：已经配置好，无需修改！**

### 第三步：触发构建

方式 1：推送代码到 main 分支
```bash
git add .
git commit -m "Setup prebuilt image"
git push origin main
```

方式 2：手动触发
1. 进入 GitHub 仓库
2. Actions → Build Prebuilt OpenClaw Image
3. Run workflow → Run workflow

### 第四步：等待构建完成

- 构建时间：约 60 分钟
- 查看进度：GitHub Actions 页面
- 构建成功后，镜像会推送到 Docker Hub

### 第五步：在 HF Spaces 使用

1. 将 `Dockerfile-hf` 重命名为 `Dockerfile`（或在 HF 设置中指定）
2. 确保 `entrypoint.sh-supabase` 在同一目录
3. 推送到 HF Spaces
4. HF 构建时间：5-10 分钟（只需拉取镜像）

## 更新流程

### 更新自定义补丁或配置
1. 修改 `patches/` 或 `characters/` 或配置文件
2. 上传到 Supabase
3. 重启 HF Space（30秒）
4. **不需要重新构建镜像**

### 更新 OpenClaw 版本或 openclaw-china
1. 修改 `openclaw-china-main/` 或 `Dockerfile-prebuilt`
2. 推送到 GitHub
3. GitHub Actions 自动构建新镜像（60分钟）
4. 重启 HF Space 拉取新镜像（10分钟）

### 更新 entrypoint.sh
1. 修改 `entrypoint.sh-supabase`
2. 推送到 HF Spaces
3. HF 重新构建（10分钟，因为只需拉取基础镜像）

## 时间对比

| 操作 | 原方案 | 预构建方案 |
|------|--------|-----------|
| 首次部署 | 60分钟 | 10分钟 |
| 修改补丁 | 60分钟 | 30秒 |
| 修改配置 | 60分钟 | 30秒 |
| 更新 OpenClaw | 60分钟 | 70分钟（60+10） |
| 修改 entrypoint | 60分钟 | 10分钟 |

## 故障排查

### 构建失败
1. 检查 GitHub Actions 日志
2. 确认 Docker Hub Secrets 配置正确
3. 确认 `openclaw-china-main` 目录存在

### HF 拉取镜像失败
1. 确认镜像已成功推送到 Docker Hub
2. 检查 `Dockerfile-hf` 中的镜像名称是否正确
3. 确认镜像是公开的（或 HF 有访问权限）

### 运行时错误
1. 检查 HF Space 日志
2. 确认 Supabase 文件已正确上传
3. 确认环境变量配置正确

## 维护建议

1. **定期更新基础镜像**：每月触发一次构建，获取最新的 OpenClaw 版本
2. **监控镜像大小**：定期清理不需要的层
3. **备份原方案**：保留 `Dockerfile-supabase` 作为备用
4. **版本标签**：使用日期标签追踪镜像版本
