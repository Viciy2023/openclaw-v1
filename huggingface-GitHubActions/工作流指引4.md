## 📋 快速开始（3步）

### 步骤 1：配置 GitHub Secrets

进入 GitHub 仓库 → Settings → Secrets → Actions

添加：
- `DOCKERHUB_USERNAME`：你的 Docker Hub 用户名
- `DOCKERHUB_TOKEN`：你的 Docker Hub Access Token

### 步骤 2：修改镜像名称

✅ **已配置完成，无需修改！**

`Dockerfile-hf` 已配置为：

```dockerfile
FROM viciy2023/openclaw-clawmate:latest
```

### 步骤 3：触发构建

```bash
git add .
git commit -m "Setup prebuilt image"
git push origin main
```

然后：
1. 等待 GitHub Actions 完成（60分钟）
2. 将 `Dockerfile-hf` 推送到 HF Spaces（重命名为 `Dockerfile`）
3. 等待 HF 构建完成（10分钟）

## 📦 预构建镜像包含

### ✅ 包含（不常变化）

- OpenClaw 官方代码 + 依赖 + 构建产物
- ClawMate 基础插件
- openclaw-china 插件（你的本地版本）
- Chromium + 所有依赖库
- agent-browser + Playwright
- agent-browser skill
- 系统依赖和工具

### ❌ 不包含（经常变化，运行时下载）

- 自定义补丁（`patches/clawmate/*.ts`）
- 角色资源（`characters/maggie/*`）
- 配置文件（`openclaw.json`, `.env`）
- 系统提示（`AGENTS.md`, `TOOLS.md` 等）
- entrypoint.sh（运行时配置逻辑）

## 🛠️ 故障排查

### GitHub Actions 构建失败

1. 检查 Secrets 配置
2. 查看 Actions 日志
3. 确认 `openclaw-china-main` 目录存在

### HF 拉取镜像失败

1. 确认镜像已推送到 Docker Hub
2. 检查 `Dockerfile-hf` 中的镜像名称
3. 确认镜像是公开的

### 运行时错误

1. 检查 HF Space 日志
2. 确认 Supabase 文件已上传
3. 确认 HF Secrets 配置正确
